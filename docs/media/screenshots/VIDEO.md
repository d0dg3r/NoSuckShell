# Promo video checklist

Use the same **demo hosts and terminal content** as in [README.md](README.md) / screenshot guide — no real credentials or IPs.

## Suggested flow (45–75 seconds)

1. App opens on main view: host list + one session.
2. Split workspace; show two terminals.
3. Open layout profiles; save or load (name visible).
4. Open settings backup section briefly (blur or fake paths if editing).
5. End on main view or split view.

## Export

- **Codec/container:** H.264 + AAC in **MP4** is widely accepted.
- **Resolution:** 1920×1080 or your monitor’s native size; avoid odd scalings.
- **Audio:** Optional quiet track or silent — some stores require audio flags; check current rules.

## Where to publish

| Channel | Action |
| --- | --- |
| GitHub README | Link: `https://github.com/…/releases/download/…/nosuckshell-demo.mp4` or YouTube |
| Microsoft Store | Partner Center → app → **Trailers** (if offered for your submission type) |
| Flathub | Optional; listings are screenshot-first — use video on website or README |

## Tools (CachyOS / Linux)

- **OBS Studio** — full control, scene switching.
- **Kooha** — quick Wayland-friendly WebM/MP4 capture.
- **wf-recorder** (Wayland) — CLI region/window recording where supported.
